`briefing_script_tester.py`

Local Windows Tkinter tool for checking narration scripts generated from existing TomeShelf backup briefings.

What it does:
- opens a backup `manifest.json` or full backup `.zip`
- lists only books whose saved briefing was generated by Perplexity
- uses the same narration prompt and Gemini script-generation call as the app audio pipeline
- lets you choose `safe` or `spoiler` mode when a fiction briefing has both variants
- shows the generated script directly in the window
- appends each generated script to a local log file for later review

Run:

```powershell
cd "C:\_George\Antigravity Playground\BookOrganizer"
py -3 .\tools\briefing_script_tester.py
```

If `py` is not available, use your local Python launcher instead.

API key:
- enter it in the GUI, or
- set `GEMINI_API_KEY`

Notes:
- this tool does not synthesize audio; it only generates the narration script
- it filters to briefings whose saved `model` contains `perplexity`
- it uses `gemini-2.5-pro` first for script generation and falls back to `gemini-2.5-flash` on HTTP 400, matching the app behavior
- the log file defaults to `tools/briefing_script_tester_log.txt` and can be changed from the GUI
