Everything TomeShelf can do.

Adding Books
  • Read-only experience — the shared page is for browsing, reading, listening, and viewing photos; viewers cannot edit, add, remove, or reorganize books
  • Shared shelves and shared books — a shared shelf opens a browsable collection; a shared book link opens directly to one book's detail view
  • Responsive layout — shelves and book detail views adapt to phones and desktop, including a mobile back path and a wider reading mode on larger screens
  • Lightbox viewing — cover images and shared additional photos open in a lightbox, with gallery navigation when more than one image is available
Managing Your Library
  • Shelf browsing — shared shelves support search and sorting so viewers can find books quickly inside a collection
  • Book detail fields — shared books can show title, author, year, publisher, edition, ISBN, subjects, notes, cover art, and additional photos depending on what the owner included
  • Photo-aware browsing — viewers can sort by books that include additional photos and open those images directly from the shared detail page
AI Research Briefings
  • Shared books can display the owner’s existing briefing in a structured reading view
  • Briefings use genre-aware structure for fiction, non-fiction, and reference titles
  • Fiction briefings default to spoiler-safe summaries, character notes, analysis, and discussion questions
  • Viewers do not regenerate or edit briefings from the shared page; they only read what the owner has already made available
Wikipedia Lookup
  • Available on every book’s detail page — one tap opens an in-app modal showing the article summary, thumbnail, and Wikidata description
  • Pass A — direct Wikipedia REST lookup verified against the book type and author name; fast, no AI cost
  • Pass B — if Pass A finds nothing, a Gemini Cloud Function identifies the exact Wikipedia article title for the book and for its author
  • Shows the book’s article when one exists; falls back to the author’s article with a notice; offers a “Search Wikipedia” link if neither is found
  • Pass B is opt-in when sharing — shelf owners choose whether viewers can trigger AI Wikipedia lookup
Sharing
  • Generate a public, read-only share link for any shelf or any individual book — viewers need no account or sign-in
  • One active link per shelf and one active link per book; any number of people can use the same link simultaneously
  • Per-share controls can include personal notes, AI-powered Wikipedia lookup, audio briefings, and for single-book shares additional photos
  • Shelf shares let viewers browse the collection and open book details; single-book shares open directly to that book’s page
  • Viewers can browse books, read briefings, view Wikipedia articles, play shared audio, and open shared photos — but cannot edit, add, or remove anything
  • Revoke links from the shelf’s Share button, the book detail Share Book button, or from Account Settings at any time
Wikipedia Lookup
  • Available only when the owner enables it for the share
  • Pass A — direct Wikipedia REST lookup verified against the book type and author name
  • Pass B — if needed, a Gemini Cloud Function identifies the most likely Wikipedia article title for the book or author
  • Shows the book article when one exists; otherwise falls back to the author page or a direct Wikipedia search link
Audio Briefings
  • Available only when the owner enables shared audio for that book or shelf
  • Shared audio plays the owner’s already-generated briefing audio; viewers do not generate new audio from the shared page
  • When available, the shared page can play existing ready audio variants directly from the browser
Cloud & Account
  • The share page itself requires no viewer account or sign-in
  • Shared data is read from the owner’s published share record and stays read-only for viewers
  • Cover images, shared additional photos, and shared audio are streamed from the owner’s Firebase-hosted assets when enabled
  • Owners manage, update, or revoke share links from the main TomeShelf app, not from the shared page
Design & Mobile
  • Vintage parchment aesthetic — warm creams, golds, and ink tones; Playfair Display headings, EB Garamond body, Courier Prime labels
  • Mobile-first — bottom navigation bar on phones, full three-column layout on desktop, hardware back-button support, safe-area insets for notched phones
  • No installation required — runs in any modern browser; no app download, no extensions
